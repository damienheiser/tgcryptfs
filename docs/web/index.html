tgcryptfs

Encrypted cloud-backed filesystem with end-to-end encryption

Download View on GitHub

Features

End-to-End Encryption

AES-256-GCM encryption with Argon2id key derivation. Your data is encrypted before it leaves your machine.

FUSE Filesystem

Mount and use like any normal directory. Works with all your existing applications.

Content Deduplication

BLAKE3 content-addressed storage. Identical data is stored only once.

LZ4 Compression

Fast compression for compressible data. Only applied when beneficial.

Distributed Mode

CRDT-based multi-machine sync with automatic conflict resolution.

Snapshots

Point-in-time filesystem snapshots with easy restore.

Quick Start

# Install (macOS ARM64)
curl -L https://github.com/damienheiser/tgcryptfs/releases/latest/download/tgcryptfs-macos-aarch64 -o tgcryptfs
chmod +x tgcryptfs
sudo mv tgcryptfs /usr/local/bin/

# Initialize with your Telegram API credentials
tgcryptfs init --api-id YOUR_API_ID --api-hash YOUR_API_HASH

# Authenticate
tgcryptfs auth --phone +1234567890

# Mount
mkdir -p ~/tgcryptfs
tgcryptfs mount ~/tgcryptfs

# Use it!
cp ~/Documents/* ~/tgcryptfs/

Security Model

Password | +--> Argon2id --> Master Key | +--> HKDF --> Metadata Key (encrypts filesystem metadata) | +--> HKDF --> Chunk Keys (per-chunk encryption keys) | +--> HKDF --> Machine Key (per-machine derived key)

The cloud backend only sees encrypted blobs. It cannot read your files, see filenames, or know your directory structure.

Building from Source

# Prerequisites: Rust 1.70+, FUSE libraries
# Linux: sudo apt-get install libfuse3-dev
# macOS: brew install macfuse

git clone https://github.com/damienheiser/tgcryptfs.git
cd tgcryptfs
cargo build --release
sudo cp target/release/tgcryptfs /usr/local/bin/
